- git stash : to save uncommitted and unstaged changes in an internal memory and revert your working directory to the latest commit.


- git stash apply : to restore the most recent stashed changes back to your working directory.
- git stash apply <stash-number> : to restore a specific stashed change using its index number.



- git stash list : to display an overview of all your stashed changes along with their index numbers.
- git stash clear : to completely empty the stash and delete all saved data.



- git stash push -m "<message>" : to save changes to the stash with a descriptive message to easily identify them later.



- git stash pop : to apply the most recent stashed change to your project and remove it from the stash list at the same time.



- git stash pop <stash-number> : to apply a specific stashed change to your project and remove it from the stash list at the same time.



- git stash drop <stash-number> : to delete a specific individual entry from the stash list using its index number.



- note: git stash apply keeps the stashed data in the stash list after applying it, while git stash pop applies the changes and automatically deletes them from the stash list.
